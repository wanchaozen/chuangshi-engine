# Security policy

## Supported versions

Until version 1.0, only the latest release receives security fixes.

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting feature for this
repository. Do not include private game data, API keys, access tokens, or
personal information in the report.

Reports should include:

- affected version;
- minimal reproduction using synthetic data;
- expected impact;
- suggested mitigation, if known.

An acknowledgement should be sent within seven days. Public disclosure should
wait until a fix or mitigation is available.
